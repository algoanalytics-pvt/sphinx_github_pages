docker_tweets.news_ingestion_aws.a_intro()[source]

This file is used to search news articles using pygooglenews api, scrape the news articles and add the news into the dynamodb News table. In this file, basic data preprocessing is also done by checking the news article language and if the news contains specific user defined keywords, if the conditions are passed then the news is added to the news table.

Mansi Maurya <mmaurya@algoanalytics.com>

Checks if the scraped news articles pass certain conditions like language check and keyword check and on the basis of those results adds news articles to news table.

docker_tweets.tweets_ingestion_aws module

docker_tweets.tweets_ingestion_aws.a_intro()[source]

This file is used to search tweets using snscrape api, scrape the tweets and add the tweets into the dynamodb tweets table. In this file, basic data preprocessing is also done by filtering the scraped tweets, checking if the tweets contain minimum tokens and do not contain irrelavant words if the conditions are passed then the tweet is added to the tweet table.

CreatedBy:

Mansi Maurya <mmaurya@algoanalytics.com>

LastUpdatedDate:

07th July 2023

ChangesMadeInCode:

Add docstrings

Function to check if tweet contains minimum(min_num_tokens) number of tokens or not.

tweet_full_text (str) : scraped tweet text min_num_tokens (int) : Minimum number of tokens a tweet should have. Defined in the config table. min_num_chars (int) : Minimum number of characters that a token should have. Defined in the config table.

docker_tweets.tweets_ingestion_aws.crawl_tweets_by_keyword(Temp_Tweets_Table, ticker_tweet_keywords, ticker_tweet_exclusions, ticker_name, tweet_followersCount, tweet_minimum_tokens, tweet_minimum_chars)[source]

Main Function for crawling tweets using snscrape library

Args:

temp_tweets_table (AWS DynamoDB Table) : “Tweets” Table from DynamoDB ticker_tweet_keywords (Str) : The query string which has to be passed to Sncscrape which will return tweets about the specific company based on the keywords. ticker_tweet_exclusions (Str) : The query string which has to be passed to Sncscrape which will not return tweets about the specific company based on the exclusion string. ticker_name (Str) : The string which will be used to store tweets along with the Ticker name in DynamoDB Table. tweet_followersCount (int) : The number used to store tweets only if the follower count of editor is greater than the defined number. tweet_minimum_tokens (int) : The number used to store tweets only if the number of tokens of the tweet is greater than the defined number. tweet_minimum_chars (int) : The number used to store tweets only if the number of characters of the tweet is greater than the defined number.

docker_tweets.tweets_ingestion_aws.filter_tweet(tweet_text)[source]

Function to check if a given sentence contains at least one word or bigram or phrase from the RELEVANT_WORDS list and does not contain any word from the FILTER_OUT_WORDS list.

Args:

tweet_text (str) : scraped tweet text

Returns:

Bool: True if the sentence is relevant, False otherwise.
