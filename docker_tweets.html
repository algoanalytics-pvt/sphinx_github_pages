docker_tweets.main.a_intro()[source]

This file is the entry point to the application. It is the first file that is executed when starting the application. It is responsible for creating all the resources required for running news & tweet ingestion task.

docker_tweets.news_ingestion_aws module

docker_tweets.news_ingestion_aws.Find_url(string)[source]

Finds if a url is present in news article. This function is used for cleaning news articles from Business Standard source, as Business standard returns unnecessary content while scraping before the main article.

Args:

string (str) : The scraped article from “business standard” news source.

Returns:

element (str): First URL from the list of urls extracted from the scraped news article.

docker_tweets.news_ingestion_aws.crawl_news_by_keyword(news_table, frequency, ticker_news_keywords, ticker_symbol, longDesc_keywords, text_search)[source]

Main Function for crawling news using pygooglenews library

Args:

news_table (AWS DynamoDB Table) : “News” Table from DynamoDB frequency (str) : The time duration from which the user wants to get the news ticker_news_keywords (str) : The query string which has to be passed to google news which will return news about the specific company. ticker_symbol (str) : The string which will be used to store news along with the Ticker name in DynamoDB Table. longDesc_keywords (str) : Keywords that will be used to search them in the scraped news article text_search (str) : Value that is used to decide if to find news using text if the results returned by finding news using title is 0

LastUpdated: 04th-July-2023

docker_tweets.tweets_ingestion_aws.crawl_tweets_by_keyword(Temp_Tweets_Table, ticker_tweet_keywords, ticker_tweet_exclusions, ticker_name, tweet_followersCount, tweet_minimum_tokens, tweet_minimum_chars)[source]

Main Function for crawling tweets using snscrape library

Parameters

tweets_tableAWS DynamoDB Table

“Tweets” Table from DynamoDB

ticker_tweet_keywordsStr

The query string which has to be passed to Sncscrape which will return tweets about the specific company based on the keywords.

ticker_tweet_exclusionsStr

The query string which has to be passed to Sncscrape which will not return tweets about the specific company based on the exclusion string.

ticker_nameStr

The string which will be used to store tweets along with the Ticker name in DynamoDB Table.

tweet_followersCountint

The number used to store tweets only if the follower count of editor is greater than the defined number.

tweet_minimum_tokensint

The number used to store tweets only if the number of tokens of the tweet is greater than the defined number.

tweet_minimum_charsint

The number used to store tweets only if the number of characters of the tweet is greater than the defined number.
